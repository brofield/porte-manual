  1. About trouble shooting procedures in the repair book
    1. The trouble shooting procedure described in this book is based on the two configurations: a diagnosis procedure in the case of a diagcode output and a diagnosis procedure in the event of a defective symptom.The concept of each procedure is as follows.


      		Diagnostic content
      		Diagnosis procedure concept
      Diag code diagnosis
      		Describe the diagnosis procedure when the diameter code is output at this time
      		The output condition of the diag code is specified by the elimination method using a single inspection or diagnostic device.
      Diagnosis by defective symptoms
      		The diagnosis procedure is described if the diameter code is not output but the defect symptoms have occurred at this time.
      		The cause of the defective symptoms, except for the conditions in which the diag code outputs, is specified by the elimination method using a single inspection and diagnostic device.
    2. There are many computers that are difficult to check alone due to the advanced and complicated vehicle -mounted system, so if the diagnostic procedure has been checked for a single check, and if there is no problem with the single inspection until the end, there is no problem.The so -called [elimination method], which identifies a computer caused by the symptoms, is incorporated into the concept of the diagnosis procedure.
    3. It is most important to listen to your offer (interview table), reproduce and confirm the bug, and then start diagnosis.In the case of the diagnosis, the defect symptoms cannot be confirmed continuously, and even if the diagnosis procedure described in this book is performed, there will be no problems and the computer will arrive, and in some cases the problem will not be resolved.There is a case.
    4. In addition, in this book, the diagnosis procedure is not expanded indefinitely, so it is assembled on the premise that duplicate failure (two or more problems) did not occur in one defect.
    5. In this book, it describes the separation of defective parts such as units, computers, and wire harness.Therefore, if there is a problem with the wire harness as a result of the diagnosis, it is necessary to perform not only the connection portion such as the unit and the computer, but also the intermediate connector connected in the middle of the wire harness.

          1. How to use the diagnostic tool TASCAN

            1. Connection with vehicles


              1. Connect Tascan to the DLC3 (data link connector No.3) installed near the driver's seat foot.

            2. How to use the diagnostic tool TASCAN


              1. When the power switch of the diagnostic tool TASCAN is turned on, the menu screen is displayed.

              2. Select the items you want to implement arbitrarily, and then proceed to work according to the instructions on the screen.
